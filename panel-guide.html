Using the OpsLink Control Panel

OpsLink Hosting uses a custom deployment of Pterodactyl, a powerful and secure control panel for managing game servers, bots, and voice hosting. This guide walks you through the main areas of your panel.

üîπ Panel URL: https://panel.opslinkcad.com
üîπ Use your login credentials provided in your order email.

1. Dashboard Overview

When you log in, your main dashboard shows all servers assigned to your account. Each tile includes:

  • Server name and short description
  • Current status (online/offline)
  • Live resource statistics (CPU, RAM, Disk)

Click any server tile to open its full management interface.

2. Console Access

The Console tab lets you view live server logs, send commands, and monitor the real-time output of your service.

  • Use the text box at the bottom to send commands (e.g. restart or game commands).
  • If the console freezes, use the Reconnect button to refresh the live socket.
  • Always check the console output before starting or stopping your server to ensure safe operations.

3. File Management & SFTP

You can edit and upload files directly through the panel‚Äôs file manager or use external SFTP access for full control.

4. Startup Configuration

The Startup tab allows you to adjust environment variables and launch parameters specific to your game or bot. You can:

  • Update tokens, API keys, or configuration flags
  • Change resource paths or arguments
  • Restart the service afterward to apply updates

5. Backups

If your plan includes backups, they‚Äôll appear under the Backups tab. From here you can:

  • Create manual backups before updates
  • Restore older versions of your files
  • Download snapshots to your computer

Backups are stored on external NVMe storage for reliability. OpsLink staff can also restore a server snapshot for you upon request.

6. Task Scheduling

You can automate certain actions under the Schedules tab ‚Äî perfect for nightly restarts or timed updates.

  • Create a schedule with your desired frequency
  • Choose an action (e.g. restart, backup, or custom command)
  • Click Enable to activate it

7. Resource Monitoring

The right-hand sidebar shows live graphs of CPU, RAM, and network activity. High spikes over long periods can indicate misbehaving scripts or excessive load.

If you frequently hit your limits, contact OpsLink support for an upgrade recommendation.

8. Security Notes

  • Do not share your panel login credentials with anyone.
  • Always log out after finishing work on public computers.
  • Never install unsigned or suspicious third-party scripts without verifying their source.
